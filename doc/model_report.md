# TikaML 模型进展报告

> 截至 2026-03-11

## 1. 项目目标

基于 Opta 事件级数据，构建足球比赛预测模型。核心输出为 7x7 比分概率矩阵，评估指标为 RPS (Ranked Probability Score)，越低越好。

**目标参考**: 博彩公司 RPS ~ 0.185

---

## 2. 当前模型架构

```
原始数据 (Opta API)
    ↓
build_features.py (8步管道)
    ├─ Step 1-4: 比赛结果 + matchstats + xG + 事件特征
    ├─ Step 5: 合并
    ├─ Step 6: 滚动特征 (EWM, 场地分割, 衍生)
    ├─ Step 7: 上下文特征 (休息天数, H2H, 积分榜, 动量, 平局率)
    └─ Step 8: 博彩赔率隐含概率
    ↓
features.csv (21,121 场 × 212 列)
    ↓
LGBMPoissonModel (src/lgbm_poisson.py)
    ├─ LightGBM Poisson 回归 × 2 (λ_home, λ_away)
    ├─ 71 个特征, Optuna 调优超参数
    ├─ Dixon-Coles ρ=-0.108 低分修正
    └─ Temperature T=0.90 概率锐化
    ↓
7×7 比分概率矩阵 → 1x2 概率
```

### 2.1 特征列表 (71个)

| 类别 | 特征 | 数量 |
|------|------|------|

| 滚动状态 | goals rolling + conceded | 4 |

### 2.2 数据规模

- 5 大联赛 (EPL, La Liga, Serie A, Bundesliga, Ligue 1)
- 12 个赛季 (2014-2025, 部分 2025-2026)
- 21,121 场比赛
- 赔率覆盖: 2018+ 赛季 ~77%, 整体 ~51%

## 3. 性能评估

### 3.1 前向链式验证 (Forward-Chain)

训练集: 该赛季之前所有赛季 (≥2016-2017)，最后一个赛季做验证。

| 赛季 | 比赛数 | RPS | 准确率 | 高置信(≥60%) Acc |
|------|--------|-----|--------|-----------------|
| 2022-2023 | 1,827 | 0.2021 | 53.1% | 68.0% |
| 2023-2024 | 1,752 | 0.1908 | 54.2% | 71.1% |
| 2024-2025 | 1,750 | 0.1974 | 53.5% | 67.5% |
| 2025-2026 | 1,290 | 0.1990 | 52.4% | 70.3% |
| **4赛季平均** | | **0.1973** | **53.3%** | **69.2%** |

### 3.2 概率校准

| P(H) 预测区间 | 实际主胜率 | 偏差 |
|--------------|-----------|------|
| 35-45% | 40.5% | -0.6% |
| 45-55% | 52.0% | -2.2% |
| 55-65% | 60.2% | -0.3% |
| 65-75% | 72.1% | -2.4% |
| 75-90% | 87.0% | -7.9% |

整体校准良好，极端区间略保守 (已通过 T=0.90 部分修正)。

### 3.3 RPS 损失分解

| 实际结果 | 占比 | 平均 RPS | 贡献占比 |
|----------|------|----------|---------|
| 主胜 | 44% | 0.175 | 39% |
| 平局 | 25% | 0.159 | 20% |
| 客胜 | 31% | 0.267 | 41% |

客胜是最大损失来源，主要因为爆冷场景 (P(H)>55% 但客胜) 的单场惩罚极高。

## 4. 优化历程

### 4.1 有效的优化

| 阶段 | RPS | 改善 | 关键改动 |
|------|-----|------|---------|
| Dixon-Coles 基线 | 0.2060 | — | 基础泊松模型 |
| + LightGBM 68特征 | 0.1997 | -0.006 | Optuna 调优, 滚动特征 |
| + 场地分割+衍生特征 | 0.1993 | -0.001 | venue rolling, xg_overperformance |
| + 赔率隐含概率 | 0.1979 | -0.001 | Pinnacle/B365 odds → 3特征 |
| + Temperature T=0.90 | 0.1976 | -0.000 | 概率锐化修正保守偏差 |
| + 比赛重要性 + 阵容变动 | **0.1973** | -0.000 | 13个新特征 (84总) |

**总改善: 0.206 → 0.197, 降幅 -0.009**

### 4.2 已尝试但无效的优化

| 方法 | RPS 变化 | 原因 |
|------|---------|------|
| Isotonic 概率校准 | +0.001 | 校准集太小，过拟合 |
| Stacking (DC+LGBM+Elo → Meta) | -0.000 | LGBM 已主导，其他基模型贡献极小 |
| LightGBM 5x Bagging | -0.000 | 微小且不稳定 |
| 时间加权训练 | +0.000 | 更多数据 > 时间衰减 |
| 双变量泊松 λ₃ | ±0.000 | λ₃ 跨赛季不稳定 (0.024 → 0.000 → 0.269) |
| 神经网络团队嵌入 | -0.000 | NN+LGBM ensemble 仅 -0.0001 |
| 比分矩阵后处理 | ±0.000 | draw boost, adaptive ρ, 1x2调整均无效 |
| 平局专家分类器 | ±0.000 | P(draw) 实际平局=0.270 vs 非平局=0.269，完全无法区分 |
| 分箱/Isotonic 后校准 | +0.000 | 校准集噪声大，精细校准不稳定 |

## 5. 已知不足

### 5.1 平局预测 = 0

**现象**: 模型从不将平局作为最大概率预测。实际平局占 25%。

**原因**: 泊松分布的数学必然性。当 λ ≥ 0.89 时 (现实比赛必然如此)，P(draw) 永远 < max(P(H), P(A))。

**影响**: 对 RPS 影响有限。模型的 P(draw) 整体校准准确 (25.6% vs 实际 25.3%)，只是无法识别「哪些场次」会平局。平局的发生主要由比赛中的随机因素 (红牌、伤停、VAR) 决定，赛前特征几乎无法捕捉。

### 5.2 高置信度区间偏保守

**现象**: P(H) 75-90% 区间实际主胜率 87%，偏低约 8%。

**影响**: Temperature T=0.90 部分修正，但残余偏差仍存在。更激进的校准会在某些赛季过拟合。

### 5.3 爆冷感知不足

**现象**: 强队主场爆冷 (拜仁负波鸿、巴萨负拉斯帕尔马斯等) 单场 RPS > 0.70。

**原因**: 爆冷由赛前不可观测因素驱动 (伤病、动力差异、战术博弈)。赔率特征已部分缓解，但仍有差距。

### 5.4 与博彩公司的差距

| | RPS |
|--|-----|
| 我们 | 0.198 |
| 博彩公司 | ~0.185 |
| 差距 | +0.013 |

差距来源:
1. **实时信息** (~60%): 伤病、阵容、转会窗、球员状态 — 博彩公司有专人跟踪
2. **市场智慧** (~25%): 大量投注者的集体判断
3. **模型结构** (~15%): 泊松分布对低分比赛建模偏差、平局无法识别

## 6. 潜在改进方向

### 6.1 高可行性

| 方向 | 预期收益 | 难度 | 说明 |
|------|---------|------|------|
| **实时阵容/伤病数据** | -0.002~0.004 | 中 | 接入 Opta squads 端点，赛前获取实际首发 (当前用历史轮换率估算) |
| **赔率实时接入** | -0.002~0.003 | 低 | 当前用历史赔率文件，改为 API 实时获取即时赔率 |
| **更多历史赛季** | -0.001 | 低 | 扩展到 2010-2014 赛季增加训练量 |
| **联赛特定参数** | -0.001 | 低 | 五大联赛分别调优 ρ 和 temperature |

### 6.2 中等可行性

| 方向 | 预期收益 | 难度 | 说明 |
|------|---------|------|------|
| **赛前阵容预测模型** | -0.002~0.004 | 高 | 根据赛程密度+历史轮换模式预测首发 |
| **多任务学习** | -0.001~0.002 | 中 | 同时预测 λ_home, λ_away, ρ，共享底层表示 |
| **时序模型 (LSTM/Transformer)** | -0.001~0.002 | 高 | 直接建模球队状态的时序演化 |

### 6.3 探索性

| 方向 | 预期收益 | 难度 | 说明 |
|------|---------|------|------|
| **对抗性特征** | 未知 | 高 | 战术克制关系 (高压 vs 控球, 防反 vs 进攻) |
| **天气/场地特征** | -0.000~0.001 | 中 | 降雨、温度、人工草/天然草 |
| **裁判特征** | -0.000~0.001 | 低 | 裁判的主场倾向、红牌率、点球率 |

## 7. 推理管道

### 使用方式

```bash
source .venv/bin/activate

# 带赔率的预测 (更准确)
python scripts/predict.py --matches "Arsenal vs Chelsea" --odds 1.85 3.60 4.50

# 从 Opta 获取赛程
python scripts/predict.py --league EPL --fetch --limit 10

# 演示模式
python scripts/predict.py
```

### 输出示例

```
Arsenal vs Chelsea
λ_home=1.84  λ_away=1.15
主胜 52.2%  |  平局 25.2%  |  客胜 22.6%
预测结果: 主胜

三组比分预测:
  主胜 [52.2%]: 2-1(9.8%), 2-0(8.6%), 1-0(8.2%)
  平局 [25.2%]: 1-1(11.9%), 0-0(6.2%), 2-2(5.6%)
  客胜 [22.6%]: 1-2(6.1%), 0-1(4.7%), 0-2(3.3%)
```
